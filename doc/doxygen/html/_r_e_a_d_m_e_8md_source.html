1 # SBF - Super Bloody Fast
2 SBF is a message transport system that allows inter app communication in two different flavours:
3 * Publisher - Subscriber
4 * Requester - Replier
5 
6 The `publisher-subscriber` consists of publishers defining topics where messages are placed. Then one or more subscribers might register to a concrete topic to read the messages that were delivered (one to many communication).
7 On the other hand, the `requester-replier` communication is about defining a channel where messages are sent from one requester to one publisher (one to one communication).

11 ## Publisher - subscriber example
12 Let's see how to create a publisher-subscriber communication.
13 
14 ### Publisher code
15 Here are the first lines you will need to create your publisher:
16 #### Create the middleware
17 ```c
18 #include "sbfCommon.h" // Common stuff such as queues, threading, ...
19 #include "sbfMw.h" // The middleware API
20 
21 ...
22 
23 // Initialise the middleware by defining some properties like the handler (e.g. tcp, udp) and the connection interface (e.g. eth0).
24 sbfKeyValue properties = sbfKeyValue_create ();
25 sbfKeyValue_put (properties, "transport.default.type", "udp");
26 sbfKeyValue_put (properties, "transport.default.udp.interface", "eth0");
27 sbfMw mw = sbfMw_create (log, properties);
28 ```
29 #### Create a queue, the connection port and a thread to dispatch events
30 ```c
31 static void*
32 dispatchCb (void* closure)
33 {
34  sbfQueue_dispatch (closure);
35  return NULL;
36 }
37 
38 ...
39 
40 sbfThread t;
41 sbfQueue queue = sbfQueue_create (mw, "default");
42 sbfThread_create (&t, dispatchCb, queue);
43 sbfTport tport = sbfTport_create (mw, "default", SBF_MW_ALL_THREADS);
44 ```
45 #### Create the publisher
46 ```c
47 static void
48 timerCb (sbfTimer timer, void* closure)
49 {
50  printf ("%u" SBF_EOL, gPublished);
51  fflush (stdout);
52 
53  gPublished = 0;
54 }
55 
56 ...
57 
58 sbfTimer_create (sbfMw_getDefaultThread (mw),
59  queue,
60  timerCb,
61  NULL,
62  1);
63 sbfPub pub = sbfPub_create (tport, queue, "topic", NULL, NULL);
64 ```
65 
66 You can find the whole example of the publisher [here](https://gitlab.com/neueda/sbf/tree/master/tools/publisher)

70 #### Create the middleware

101 #### Create the subscriber
102 ```c
103 static void
104 timerCb (sbfTimer timer, void* closure)
105 {
106  printf ("%u low=%llu average=%llu high=%llu" SBF_EOL,
107  gMessages,
108  gTimeLow == UINT64_MAX ? 0 : (unsigned long long)gTimeLow,
109  gMessages == 0 ? 0 : (unsigned long long)gTimeTotal / gMessages,
110  (unsigned long long)gTimeHigh);
111 }
112 
113 static void
114 messageCb (sbfSub sub, sbfBuffer buffer, void* closure)
115 {
116  uint64_t* payload = sbfBuffer_getData (buffer);
117 }
118 ...
119 
120 sbfTimer_create (sbfMw_getDefaultThread (mw),
121  queue,
122  timerCb,
123  NULL,
124  1);
125 sbfPub pub = sbfPub_create (tport, queue, "topic", NULL, NULL);

137 ## Vagrant
138 If you are running on OS X or Windows don't worry, we provide a vagrant file to create a VirtualBox ubuntu image including all the necessary stuff to run.
139 You will need to download [VirtualBox](https://www.virtualbox.org/) and [Vagrant](https://www.vagrantup.com/).
140 1. Unzip ubuntu.zip folder
141 `user@host:~$ unzip ubuntu.zip`
142 2. Change your directory to ubuntu.
143 `user@host:~$ cd ubuntu`
144 3. Edit the `synced_folders` file and replace the `hostpath` pointing to your shared folder (this will mount the directory under /mnt/host/).
145 4. Run vagrant
146 `user@host:~/ubuntu$ vagrant up`
147 5. Now you are ready to run the Ubuntu image on VirtualBox
148 6. You will be asked to prompt your username and password
149 `username:vagrant`
150 `password:vagrant`

156 ## Build the SBF
157 Build the SBF by running the following commands:
158 ```
159 cd sbf
160 mkdir build && cd build
161 cmake ..
162 make build && make install
163 ```
164 You will find the binaries at `cd build/install/bin`.
165 ```
166 sbf-replier
167 sbf-requester
168 sbf-publisher
169 sbf-subscriber
170 ```
171 The requester sends messages and the replier reads the message and replies.
172 Something similar for the publisher and subscriber.
